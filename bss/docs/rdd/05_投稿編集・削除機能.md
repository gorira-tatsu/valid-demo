# 05. 投稿編集・削除機能

## 目的
- 投稿者本人が投稿内容を修正または削除できるようにする。

## 要件
- 投稿者は編集用キーを入力して投稿を編集できる
- 投稿者は編集用キーを入力して投稿を削除できる
- 編集後は更新日時を更新する
- 削除後は一覧および詳細から非表示にする

## 入力項目
- 編集時:
  - タイトル
  - 本文
  - 投稿者名
  - 編集用キー
- 削除時:
  - 編集用キー

## 業務ルール
- 編集用キーが一致しない場合は更新、削除を許可しない
- 削除は論理削除とする
- 削除済み投稿に対する再編集は不可とする
- 更新成功後は詳細画面で更新内容と更新日時を確認できる
- 削除成功後は一覧、詳細、コメント投稿の各導線で同じく再操作不可にする
- 更新成功後は同一投稿の詳細画面に留まり、成功結果のみを表示する
- 削除成功後は対象投稿の詳細や編集画面に留まらず、一覧または同等の非表示状態へ遷移する

## 画面仕様
- 削除操作前に確認ダイアログを表示する
- 編集画面では現在の投稿内容を初期表示する
- 編集失敗時は入力内容を保持して再表示する
- 削除成功後は対象投稿の編集画面へ戻さない

## API 仕様案

### `PUT /posts/:postId`
- 概要: 投稿を更新する
- リクエスト項目:
  - `title`
  - `body`
  - `authorName`
  - `editKey`

### `DELETE /posts/:postId`
- 概要: 投稿を削除する
- リクエスト項目:
  - `editKey`

## バリデーション
- 更新時の入力条件は新規投稿と同一
- 編集用キー不一致時は「編集用キーが正しくありません」を表示する

## 例外系
- 対象投稿が存在しない場合は 404 を返す
- 既に削除済みの場合も 404 を返す
- 削除確認ダイアログを経由しない削除成立は許可しない
- 編集用キー不一致後は入力内容を保持したまま再入力できる
