# 02. 投稿一覧機能

## 目的
- 利用者が最新の投稿を素早く確認できるようにする。

## 要件
- 未削除の投稿を新しい順で一覧表示する
- 一覧から各投稿の詳細画面へ遷移できる
- 一覧画面から新規投稿画面へ遷移できる

## 表示項目
- タイトル
- 投稿者名
- 投稿日時
- 本文先頭の抜粋（最大 120 文字程度）
- コメント件数

## 業務ルール
- 削除済み投稿は表示しない
- 本文抜粋が上限を超える場合は末尾を省略する
- 投稿が 0 件の場合は空状態メッセージを表示する
- 一覧のコメント件数は詳細画面で閲覧できる未削除コメント件数と矛盾しない
- 一覧の既定順は `createdAt` の新しい順とし、同一時刻では `id` の降順で安定化する
- 投稿更新によって一覧の既定順は変えない

## 画面仕様
- 1 画面に 20 件まで表示する
- 20 件を超える場合はページネーションまたは「もっと見る」を提供する
- 空状態では「まだ投稿がありません」などの文言を表示する
- ページ遷移や再取得を行っても、同一投稿が同一ページ内で重複表示されない

## API 仕様案

### `GET /posts`
- 概要: 投稿一覧を取得する
- クエリ:
  - `page`: ページ番号。省略時は 1
  - `limit`: 取得件数。省略時は 20
- レスポンス項目:
  - `items`: 投稿配列
  - `page`
  - `limit`
  - `totalCount`
- 境界条件:
  - `page < 1` は `400`
  - `limit < 1` は `400`
  - `limit > 20` は `400` または 20 に丸めてもよいが、実装全体で統一する
  - 総ページ数超過の `page` は空配列を返す

## 例外系
- 取得失敗時は再読み込み導線を表示する
- 不正なページ番号指定時は 400 を返す
- 削除直後に一覧を再表示した場合、対象投稿は再出現しない
- 取得失敗から再試行で回復した後は stale な失敗表示を残さない
