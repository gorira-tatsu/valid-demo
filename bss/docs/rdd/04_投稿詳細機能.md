# 04. 投稿詳細機能

## 目的
- 利用者が投稿本文とコメントを読めるようにする。

## 要件
- 投稿の全文を表示する
- 投稿に紐づく未削除コメントを表示する
- 詳細画面からコメント投稿、投稿編集、投稿削除へ遷移または操作できる

## 表示項目
- タイトル
- 本文
- 投稿者名
- 投稿日時
- 更新日時
- コメント一覧

## 業務ルール
- 投稿が存在しない場合は 404 画面を表示する
- 更新日時は未更新の場合は表示しない
- コメントは古い順で表示する
- 削除済み投稿は URL を直接指定しても 404 と同等に扱う
- 詳細画面で表示するコメント件数と内容は、直前の成功したコメント投稿結果を反映する
- 同一時刻のコメントは `id` の昇順で安定化する

## 画面仕様
- 本文の改行を保持する
- コメント 0 件の場合は空状態メッセージを表示する
- 編集、削除操作は投稿エリア近傍に配置する
- 更新済み投稿では作成日時と更新日時を判別できる表示にする

## API 仕様案

### `GET /posts/:postId`
- 概要: 投稿詳細を取得する
- レスポンス項目:
  - `post`: 投稿情報
  - `comments`: コメント配列

## 例外系
- 削除済み投稿は 404 と同等に扱う
- 取得失敗時はエラーメッセージと一覧へ戻る導線を表示する
- 取得失敗後に再試行できる場合、成功メッセージは残さない
- 再試行で回復した後は、更新日時表示とコメント件数整合が直前の成功状態に戻る
