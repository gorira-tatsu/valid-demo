# 00. 前提とスコープ

## 目的
- 誰でも気軽にテキスト投稿できる簡易掲示板を提供する。
- 小規模なコミュニティや検証用途で利用できる最小構成を定義する。

## 想定ユーザー
- 投稿を読むだけの閲覧者
- 投稿を作成する利用者
- 投稿に対してコメントする利用者

## MVP スコープ
- 投稿を時系列で一覧表示できること
- 投稿の詳細を閲覧できること
- 投稿を新規作成できること
- 投稿者本人が編集、削除できること
- 投稿にコメントできること

## スコープ外
- 会員登録、ログイン、ログアウト
- 画像、動画、ファイル添付
- タグ、カテゴリ、全文検索
- いいね、リアクション、通報
- 管理者承認、NG ワード管理、監査ログ
- 通知、メール送信、外部連携

## 前提条件
- 投稿者本人確認は、投稿作成時に利用者自身が入力して設定する編集用キーで代替する。
- システムは編集用キーの平文を再表示しない。利用者は作成時に自分で保管する。
- 投稿、コメントはテキストのみ扱う。
- 日本語 UI を前提とする。

## 成功条件
- 初見の利用者が説明なしで投稿、閲覧、コメントまで完了できること
- 不正な入力に対して、明確なエラーメッセージが返ること
- 削除済みデータが一覧や詳細に表示されないこと
