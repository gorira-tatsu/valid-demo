# 06. コメント機能

## 目的
- 利用者が投稿に対して意見や返信を書き込めるようにする。

## 要件
- 投稿詳細画面からコメントを投稿できる
- 投稿に紐づくコメント一覧を表示できる
- コメント投稿後は同一詳細画面上に反映される

## 入力項目
- コメント本文
- 投稿者名

## 業務ルール
- 投稿者名未入力時は `名無しさん` を設定する
- 削除済み投稿にはコメントできない
- コメントは投稿作成日時の古い順で表示する
- コメント投稿成功時は一覧のコメント件数にも整合した状態で反映される
- コメント作成成功時は `id` と `createdAt` が確定し、直後の再取得でも同一順序で返る

## 画面仕様
- コメントフォームは投稿詳細画面下部に表示する
- コメント送信成功後はフォームを初期化する
- コメント 1 件ごとに投稿者名、本文、作成日時を表示する
- コメント送信失敗時は入力内容を保持し、同一詳細画面上で再送可能にする
- コメント送信中は同じ入力の重複送信を成立させない

## API 仕様案

### `POST /posts/:postId/comments`
- 概要: コメントを投稿する
- リクエスト項目:
  - `body`
  - `authorName`
- レスポンス項目:
  - `id`
  - `postId`
  - `body`
  - `authorName`
  - `createdAt`

## バリデーション
- 本文未入力時は「コメントを入力してください」
- 本文 1,000 文字超過時は上限超過エラーを表示する

## 例外系
- 投稿が存在しない、または削除済みの場合は 404 を返す
- 保存失敗時は入力内容を保持してエラーを表示する
- 削除直後の投稿に対するコメント送信は成功させない
- 再試行で回復した後は、失敗時メッセージを残さない
- 成功後に stale な失敗メッセージを残さない
