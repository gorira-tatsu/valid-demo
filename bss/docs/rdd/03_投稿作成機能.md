# 03. 投稿作成機能

## 目的
- 利用者が新しい話題を掲示板に投稿できるようにする。

## 要件
- タイトル、本文、投稿者名、編集用キーを入力して投稿できる
- 投稿成功後は投稿詳細画面へ遷移する
- 入力不備がある場合は保存せずにエラー表示する

## 入力項目
- タイトル
- 本文
- 投稿者名
- 編集用キー

## 業務ルール
- 投稿者名が未入力の場合は `名無しさん` を設定する
- 編集用キーは利用者が作成時に入力して設定する
- 編集用キーは確認入力なしでもよいが、利用者に保管を促す
- 投稿成功後に作成日時を記録する
- 送信中に同じ投稿内容を重複送信しても、二重作成は成立させない
- 投稿成功後は `id` と `createdAt` が確定し、直後の再取得でも同じ投稿を参照できる

## 画面仕様
- 必須項目には明示表示を行う
- 本文入力欄は複数行テキストエリアとする
- 投稿完了時に「投稿が完了しました」を表示してもよい
- 投稿完了後は投稿詳細へ遷移し、成功メッセージを表示する場合は一度の成功に対して一つだけ表示する

## API 仕様案

### `POST /posts`
- 概要: 新規投稿を作成する
- リクエスト項目:
  - `title`
  - `body`
  - `authorName`
  - `editKey`
- レスポンス項目:
  - `id`
  - `title`
  - `body`
  - `authorName`
  - `createdAt`

## バリデーション
- タイトル未入力時は「タイトルを入力してください」
- 本文未入力時は「本文を入力してください」
- 編集用キー未入力時は「編集用キーを入力してください」
- 最大文字数超過時は各項目ごとの上限超過メッセージを表示する

## 例外系
- 保存失敗時は入力内容を保持したままエラーを表示する
- 二重送信防止のため、送信中は投稿ボタンを非活性にする
- 再試行可能な失敗から回復した後のみ、再度送信操作を受け付ける
- 成功後に stale な失敗メッセージを残さない
